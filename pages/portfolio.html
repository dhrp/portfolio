Portfolio website

Single Page Web Application

Design

The idea behind the interaction of this site is to encourage users to 'discover' items in the portfolio. By showing only a visual hint and by providing the user control over what topics to look at a visitor explores the site rather then 'goes through it'.

Technology

Reasoning behind making this website was not only to give this portfolio a general facelift but also to showcase some technologies.

Amongst other things it demonstrates the power of Ember.js to create a dynamic application completely in Javascript. This means that as you go through the site, the different pages are all composed in your browser, rather than by any server side scripts.

Gracefull degredation

In case the user (or search engine) does not support Javascript we still provide a decent fallback site. Please have a look at the HTML source. Initially each portfolio item is represented as a regular div with a regular link.

When your browser supports it it parses all the items from the regular HTML page and replaces them with a Ember view. This has the benefit that we can easily manipulate which items are shown depending on user input.

Requirements

Some of the requirements which were set:

  • Allows portfolio items to be in multiple categories
  • Is search engine friendly (progresssive enhancement)
  • Works well on mobile
  • is a discovery process more than a static website
